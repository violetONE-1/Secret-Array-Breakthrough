## 题目功能描述

本题目来源于综艺《最强大脑》第 13 季《密阵突围》。游戏在 25×25 的矩形网格上进行，每个格子随机包含一个字母（A~Z）和一个数字（0~9）。玩家选择 5 个起始格子后，可将相邻且满足合并条件（字母相同数字不同，或数字相同字母不同）的两格合并：中心格保留并累加对应属性（数字相加取个位，字母相加循环进位），邻格清空。最终目标为尽可能多地合并格子、优化步数与用时。

**Level 1（必做）：**
1. 控制台网格界面，支持键盘交互
2. 实现完整的合并规则判定与盘面更新
3. 对操作进行正误判断并给出提示
4. 内置 ≥3 道题面，支持选题作答
5. 记录玩家用时、正确率、得分，形成排行榜

**Level 2（选做）：** 图形化界面（SFML）、自定义盘面大小、用户自创题面并保存、回放模式、简单 AI 对手

**Level 3（选做）：** 动画与音效、操作提示、高级 AI（极小化极大搜索）、闯关模式

## 二、设计思路

### 2.1 系统功能模块划分

系统采用 **Model-View-Controller (MVC)** 架构，自顶向下分为四个模块：

| 模块 | 职责 | 核心类 |
|------|------|--------|
| **核心逻辑模块** (Model) | 25×25 网格数据表示、合并规则纯算法、游戏状态快照与操作记录 | `Cell`, `Grid`, `MergeRule`, `GameState`, `Move` |
| **数据持久化模块** (Model 子层) | 题库加载/保存、排行榜读写、回放录制/导出，全部使用纯文本文件 | `Puzzle`, `PuzzleManager`, `ScoreRecord`, `Leaderboard`, `ReplayBuffer`, `FileManager` |
| **交互渲染模块** (View) | 统一的 `IRenderer` 抽象接口，支持控制台 ASCII 和 SFML 图形两种实现；输入坐标转换 | `IRenderer`, `ConsoleRenderer`, `SFMLRenderer`, `InputHandler` |
| **流程控制模块** (Controller) | 游戏主循环调度、阶段切换（菜单→选题→游戏→结算→排行榜）、AI 决策 | `GameController`, `Player`, `AIPlayer` |

**协同流程：** 用户输入 → Controller 校验（调用 MergeRule）→ 更新 Model（Grid + GameState）→ 通知 View 重绘（IRenderer::render）

### 2.2 类体系设计

共计 18 个核心类，核心类的主要数据与功能如下：

| 类名 | 主要数据成员 | 主要成员函数（功能） |
|------|------------|-------------------|
| **Cell** | `char _letter`, `int _number`, `bool _empty`, `bool _selected` | `sameLetter(Cell&)` / `sameNumber(Cell&)` 比较属性；`toString()` 格式化显示 |
| **Grid** | `int _rows, _cols`, `vector<vector<Cell>> _cells` | `mergeCells()` 合并两格；`findAllValidMoves()` 扫描合法操作；`serialize()` / `deserialize()` 序列化 |
| **MergeRule** | （纯静态类，无成员） | `canMerge(a,b)` 判定合法性；`mergeLetters(a,b)` / `mergeNumbers(a,b)` 合并算法；`getMergedCell(a,b)` 返回合并结果 |
| **GameState** | `Grid _grid`, `int _stepsTaken`, `time_point _startTime`, `vector<Move> _moveHistory` | `recordMove()` 记录操作；`elapsedSeconds()` 计时；`calculateAccuracy()` 正确率；`checkGameOver()` 胜负判断 |
| **Move** | `int _srcRow/Col, _dstRow/Col`, `Direction _dir`, `char _resultLetter`, `int _resultNumber` | `serialize()` / `deserialize()` 与回放文件互转 |
| **Puzzle** | `string _id, _name`, `int _gridSize`, `Grid _initialGrid` | `serialize()` / `deserialize()` 与题面文件互转 |
| **PuzzleManager** | `vector<Puzzle> _puzzles`, `string _puzzleDir` | `loadBuiltin()` / `loadCustom()` 加载题面；`findById()` 按 ID 检索 |
| **ScoreRecord** | `string _playerName, _puzzleId`, `double _timeSeconds, _accuracy`, `int _steps, _score` | `calculateScore()` 自动计算：`accuracy×1000 + max(0,500-time×2) + steps×10` |
| **Leaderboard** | `vector<ScoreRecord> _records`, `string _filePath` | `add()` 追加记录并保存；`topByScore(n)` / `topByTime(n)` 多维度排序 |
| **ReplayBuffer** | `string _puzzleId`, `vector<Move> _moves`, `bool _recording` | `record()` 录制操作；`saveToFile()` / `loadFromFile()` 导出导入 |
| **FileManager** | `string _dataRoot` | `readAll()` / `writeAll()` / `appendLine()` 统一封装文件 I/O |
| **IRenderer** | （纯虚接口，无成员） | `render(GameState&)=0`、`showMenu()=0`、`showLeaderboard()=0`、`handleEvents()=0` |
| **ConsoleRenderer** | `int _cellWidth` | 终端 ASCII 网格绘制 + `_getch()` 键盘输入 |
| **SFMLRenderer** (Lv2+) | `sf::RenderWindow _window`, `sf::Font _font`, `float _cellSize` | SFML 窗口网格绘制 + `sf::Event` 鼠标/键盘事件 |
| **InputHandler** | `float _cellSize, _gridX, _gridY` | `convertPixelToGrid(px,py)` 像素→网格坐标转换 |
| **GameController** | `unique_ptr<GameState>`, `unique_ptr<PuzzleManager>`, `unique_ptr<Leaderboard>`, `unique_ptr<IRenderer>` 等 | `run()` 主循环；`selectPuzzle()` 选题；`executeMerge()` 执行合并；`submitAnswer()` 结算 |
| **Player** | `string _name`, `vector<pair<int,int>> _startingCells` | 玩家名称与 5 个起始点管理 |
| **AIPlayer** (Lv2+) | `AIDifficulty _difficulty` | `selectMove(Grid&)` 策略选择（随机/贪心/搜索） |

**类间关系：** `IRenderer` 为抽象基类，`ConsoleRenderer` 和 `SFMLRenderer` 公有继承；其余类通过 `unique_ptr`（独占所有）或值语义（组合）关联，GameController 为中枢。

### 2.3 数据文件设计

全部使用**纯文本 `.txt` 格式**，不引入数据库或第三方序列化库。

| 数据类型 | 文件路径 | 读写时机 | 格式说明 |
|---------|--------|---------|---------|
| **题面** | `data/puzzles/builtin_*.txt` | 启动加载 / 用户创建时保存 | 首行为"行数 列数"，后续为格子数据（每格"字母+数字"），末行为起点坐标列表 |
| **排行榜** | `data/leaderboard.txt` | 启动加载 / 每题提交后追加 | CSV 风格，每行：`玩家名, 题面ID, 用时(s), 步数, 正确率, 总分, 时间戳` |
| **回放** | `data/replays/replay_*.txt` | 每局结束导出 / 回放时读取 | 头部为 `[Puzzle]`、`[Player]`、`[Start/End]` 等元信息，序列为 `[Move] 序号 src->dst 方向 结果` |

**题面文件示例：**
```
25 25
A1 B2 C3 ...（25×25 个格子）
5
0,0 2,3 6,10 12,15 20,20
```

**排行榜文件示例：**
```
Alice, builtin_01, 45.23, 12, 1.00, 980, 2026-05-15 14:23:10
Bob,   builtin_01, 52.10, 15, 0.93, 870, 2026-05-15 14:30:45
```
